# 智慧党建综合管理平台使用说明书

## 1. 项目组成

本项目由三个服务组成：

| 服务 | 技术 | 默认端口 | 项目目录 |
| --- | --- | --- | --- |
| 前端 | Vue 3 + Vite + Element Plus | 5173 | `smart-party-building/frontend` |
| 业务后端 | Spring Boot 3 + MyBatis-Plus + MySQL | 8080 | `smart-party-building/backend` |
| AI 助手 | FastAPI + ChromaDB + 通义千问 | 8000 | `promax/smart-party-ai-service` |

前端将 `/api` 请求代理到 Java 后端 `http://localhost:8080`。Java 后端再通过
`http://localhost:8000` 调用 AI 助手。

## 2. 环境要求

建议安装以下环境：

- JDK 17
- Maven 3.9+
- Node.js 20+
- npm 10+
- MySQL 8
- Python 3.10 或 3.11

当前电脑可使用：

- JDK：`D:\jdk17.0.17_10`
- Maven：`D:\apache-maven-3.9.11`
- MySQL：端口 `3306`

Redis 配置位于后端 `application.yml`，默认端口为 `6379`。当前核心功能未依赖
Redis，未启动 Redis 时业务后端仍可运行。

## 3. 数据库准备

后端数据库配置文件：

`smart-party-building/backend/src/main/resources/application.yml`

当前默认配置：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/example_db
    username: root
    password: 123456
```

项目当前没有完整的初始建库 SQL，仅包含部分增量 SQL。因此在新电脑部署时，应先
从已有环境导出完整数据库：

```powershell
mysqldump -uroot -p123456 --default-character-set=utf8mb4 example_db > example_db.sql
```

在新电脑创建并导入数据库：

```powershell
mysql -uroot -p123456 -e "CREATE DATABASE example_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
mysql -uroot -p123456 --default-character-set=utf8mb4 example_db < example_db.sql
```

确认 MySQL 已启动：

```powershell
Get-Service MySQL
Start-Service MySQL
```

## 4. 启动 AI 助手

进入 AI 项目目录：

```powershell
cd "C:\Users\28586\Desktop\党员\promax\smart-party-ai-service"
```

首次运行或虚拟环境失效时，重新创建虚拟环境：

```powershell
Remove-Item -Recurse -Force .venv
py -3.11 -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
```

如果 PowerShell 禁止激活脚本，可执行：

```powershell
Set-ExecutionPolicy -Scope Process Bypass
.\.venv\Scripts\Activate.ps1
```

复制并配置环境变量：

```powershell
Copy-Item .env.example .env
```

编辑 `.env`：

```env
DASHSCOPE_API_KEY=你的阿里云百炼API密钥
DISTANCE_THRESHOLD=0.6
```

知识库 PDF 放在 `knowledge` 目录。首次添加或更新 PDF 后构建索引：

```powershell
python scripts/build_index.py
```

启动 AI 服务：

```powershell
python main.py
```

首次启动会加载 `BAAI/bge-small-zh-v1.5` 模型，通常需要等待约 20 秒。日志出现
“系统已就绪”后才能正常问答。

验证地址：

- AI 状态：<http://localhost:8000/api/status>
- AI 接口文档：<http://localhost:8000/docs>
- AI 独立页面：<http://localhost:8000/>

## 7. 推荐启动顺序

每个服务使用一个独立终端窗口，按以下顺序启动：

1. 启动 MySQL。
2. 启动 AI 助手：`python main.py`。
3. 启动 Java 后端：`mvn spring-boot:run` 或使用 IDEA。
4. 启动前端：`npm run dev`。
5. 浏览器访问 `http://localhost:5173`。

## 9. 启动状态检查

检查端口是否正常监听：

```powershell
Get-NetTCPConnection -State Listen |
  Where-Object { $_.LocalPort -in 3306, 8000, 8080, 5173 } |
  Select-Object LocalPort, OwningProcess
```

正常情况下应看到：

```text
3306  MySQL
8000  AI 助手
8080  Java 后端
5173  Vue 前端
```

## 10. 停止项目

在每个运行服务的终端窗口按 `Ctrl + C`。

也可以按端口停止对应进程：

```powershell
$ports = 5173, 8080, 8000
Get-NetTCPConnection -State Listen |
  Where-Object { $_.LocalPort -in $ports } |
  ForEach-Object { Stop-Process -Id $_.OwningProcess }
```

## 11. 常见问题

### 前端出现大量 403

- 确认已经登录正确角色。
- 清除浏览器 `localStorage` 后重新登录。
- 确认 Java 后端为最新代码并已重启。

### 后端无法启动

- 确认 MySQL 已启动。
- 确认数据库名、账号和密码与 `application.yml` 一致。
- 确认使用 JDK 17，而不是其他 JDK。
- 确认端口 8080 未被占用。

### AI 助手一直显示初始化中

- 等待模型加载完成后访问 `http://localhost:8000/api/status`。
- 确认 `.env` 中已配置有效的 `DASHSCOPE_API_KEY`。
- 确认 `knowledge` 中存在 PDF，且 `vector_db` 索引已构建。
- 查看日志：`Get-Content logs/app.log -Wait`。

### 8000 端口被占用

AI 助手必须独占 8000 端口，不要同时启动其他 Django 或 FastAPI 项目：

```powershell
Get-NetTCPConnection -State Listen -LocalPort 8000
```

### 前端修改后没有生效

- 确认 `npm run dev` 正在运行。
- 浏览器按 `Ctrl + F5` 强制刷新。
